**Goal:** Add a subagent capability (pi skips it by default) via the *minimum credible* mechanism — a single LLM-callable **`spawn_agent` tool** that runs a child `AgentSession` headlessly with the **parent's model inherited**, a role/prompt + tool subset, returns the child's final text + cost, and always disposes it. On top: two **presets** — **deep-work** (droid-factory: spec→plan→execute→autoreview, bounded re-execute) and **simple** (single-pass). Offline-buildable; real QpD measured live later (user-gated).

**Architecture (ponytail — no new package/framework):** new `packages/coding-agent/src/core/subagent/` registered via the existing `customTools` seam of `createAgentSession`. Runner mirrors the PROVEN SP-1 `packages/eval/src/runner.ts` shape (createAgentSession → subscribe → Promise.race([prompt, timeout]) → getSessionStats → dispose in finally). Agents are data presets, not subclasses. Deep-work loop = plain sequential orchestration (no DAG). No Bun/Rust/Docker/YAML, NO new deps.

## Key facts (verified)
1. Subagent = fresh headless AgentSession (in-process; no subprocess/IPC).
2. Model inheritance = pass parent model into child `createAgentSession({model})`; child NEVER re-resolves a default.
3. Cost = `getSessionStats().cost` (provider-authoritative); surface child cost in the tool result `details` so parent transcript + eval meter see subagent spend.
4. Deep-work = sequential phase orchestration; autoreview = final review phase whose FAIL verdict triggers one bounded re-execute.
5. `spawn_agent` `executionMode:"sequential"` (ponytail-safe; parallelism is the deferred DAG's job).
6. **Recursion guard:** child sessions created with `enableSubagents:false` → no `spawn_agent` → no infinite delegation (asserted by test).
7. Faux proves MECHANICS only (sequencing, model-inherited, cost captured, dispose runs); agent QUALITY = live QpD A/B, user-gated.
